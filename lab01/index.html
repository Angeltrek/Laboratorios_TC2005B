Primera parte

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de computadoras interconectadas que utilizan el protocolo TCP/IP que permite la conexión y comunicación entre computadoras y redes en el mundo.

World Wide Web es el sistema de información en linea que se construye sobre la infraestructura de internet la cual sirve para enlazar recursos que están ubicados en diferentes computadoras y servidores en todo el mundo, utiliza el protocolo HTTP para la transferencia de datos.


¿Cuáles son las partes de un URL?

URL significa Uniform Resource Locator la cual es una cadena de caracteres que se utiliza para identificar recursos en la web, como páginas web, imágenes, archivos de texto, etc. Un URL se conforma de las siguientes partes:

  • Protocolo: Indica el protocolo de comunicación que se utilizará para acceder al recurso. Por ejemplo, HTTP o HTTPS para la transferencia de archivos, etc.
  • Nombre de dominio: Es el nombre que idenficia a un sitio de internet. Los nombres de dominio pueden incluir subdominios como "www", que normalmente se utiliza para indicar el servidor web.
  • Ruta: Es la ubicación específica del recurso en el servidor web, por ejemplo "www.sitio.com/pagina.html" donde "/pagina.html" es la ruta que indica el archivo especifico que está solicitando en el servidor.
  • Parámetros de consulta: Un URL puede incluirlos, los cuales sirven para personalizar la respuesta del servidor, por ejemplo, si se tiene "www.sitio.com/busqueda?q=ejemplo", donde "?q=ejemplo" es la cadena de consulta.
  • Fragmento: Opcionalmente, un URL puede incluir un fragmento que indica una ubicación especifica dentro del recurso. Se indica con el símbolo "#" seguido del identificador del fragmento, las cuales se usan en la navegación dentro de esta página.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • HTTP: Son acciones que indican qué operación se debe realizar para un recurso en particular.
  • GET: Es utilizado para obtener datos del servidor y no debe tener ningún otro efecto.
  • HEAD: Similar a GET, pero solo solicita los encabezados de respuesta, sin el cuerpo del mensaje.
  • POST: Se utiliza para enviar datos al servidor para crear un recurso nuevo. Los datos enviados con POST son generalmente enviados a través del cuerpo del mensaje HTTP.
  • PUT: Se utiliza para enviar datos al servidor para crear o remplazar el recurso en la ubicación de destino. PUT puede hacer varias solicitudes a la vez pero tiene el mismo efecto que si se hace solo una vez.
  • PATCH: Es similar a PUT, pero se utiliza para aplicar modificadores parciales a un recurso, es útil cuando solo se necesita actualizar ciertas partes de un recurso sin remplazarlo todo.
  • DELETE: Sirve para solicitar que el servidor elimine un recurso en particular.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe de usar el método POST, ya que los datos enviados a través de este método no se muestran en la URL, además, se pueden enviar grandes cantidades de datos, por lo que se pueden enviar datos sensibles a través de este método.


¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Cuando se accede a un página a través de una URL en un navegador web, se utiliza el método HTTP GET por defecto. Esto es parte del comportamiento estándar de los navegadores web al solicitar recursos a través de URLs.


Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No es ningún error, significa que la solicitud ha sido completada satisfactoriamente y que el servidor ha devuelto el contenido solicitado.


¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

El error 404 significa que el recurso solicitado no fue encontrado en el servidor. A veces puede haber enlaces rotos o eliminados, pero el responsabilidad del desarrollador garantizar que el sitio web esté disponible y que los enlaces funcionen.


¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

El error 500 ocurre cuando existe un error interno del servidor, por lo que indica que algo mal ha pasado del lado del servidor al procesar la solicitud, por lo que sigue siendo responsabilidad del desarrollador corregir el sitio web.


¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

  • center: Se utilizaba para centrar el contenido de la página.
  • font: Se utilizaba para cambiar el estilo de las fuentes.
  • frame, framset, noframes: Se utilizaban para definir marcos en una página web.
  • strike, s, u: Se utilizaban para aplicar tachado, subrayado y otros estilos.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

  • Nuevas etiquetas semánticas
  • Mejora en formularios
  • Multimedia integrada
  • Gráficos y animaciones
  • APIs y capacidades avanzadas
  • Compatibilidad con dispositivos móviles

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

  1. Idenficación de necesidades.
  2. Planificación y diseño.
  3. Implementación y desarrollo.
  4. Pruebas de validación.
  5. Despliegue e Implementación.
  6. Operación y mantenimiento.
  7. Evolución y mejora.
  8. Retiro o reemplazo.

¿Cuál es el ciclo de desarrollo de sistemas de información?

  1. Análisis de requerimientos.
  2. Diseño del sistema.
  3. Implementación y codificación.
  4. Pruebas.
  5. Despliegue e Implementación.
  6. mantenimiento.
