¿Qué es package.json?

Sirve como un registro centralizado de información sobre el proyecto, incluyendo su nombre, versión, dependencias, scripts de ejecución, autor, licencia, y más

Gestión de dependencias

Lista todas las dependencias necesarias para que el proyecto funcione correctamente.

Versionamiento del proyecto

Incluye la versión actual del proyecto, permitiendo seguir un esquema de versionamiento semántico.

Scripts de ejecución

Permite definir scripts de shell que se pueden ejecutar usando el comando npm run <nombre_del_script>.

Metadatos del proyecto

Incluye información sobre el autor del proyecto, la licencia bajo la cual se distribuye el código, la URL del repositorio, y otros metadatos relevantes.
